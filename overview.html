Choosing the Right AI for Our Event Chatbot

To make our event chatbot smart and helpful, we need to choose the best "brain" for it. Here's a look at different ways we can do this, with their good points and potential downsides, keeping our project goals and your needs in mind.

1. Using a "Home-Grown" AI on a Laptop

(Like our current experimental setup)

This means we run the AI software directly on a standard laptop. It's like having a small, dedicated helper working right here.

  • Your Data Stays With You: All information about events and user chats remains private on the laptop.
  • No Ongoing AI Fees: Once set up, there are no extra costs for the AI to answer questions.
  • Works Offline: The chatbot can still work even if there's no internet.
  • Great for Learning: Helps us understand how these AIs work from the inside out.
  • Limited Power: A laptop can only handle so much. This might mean slower answers or not being able to process very complex questions or lots of event details at once.
  • Can Be Slow: More advanced AI or lots of data can make it sluggish.
  • Needs Technical Setup: Requires some technical know-how to get it running and keep it updated.
  • Not for Heavy Use: Might struggle if many people try to use it at the same time.

2. Using School's AI Infrastructure (For Project Development)

Our school has powerful computers designed for AI tasks. We could potentially use these resources to build and test our chatbot during the project phase. This gives the AI more "horsepower" than a standard laptop.

  • Excellent for Project Stage: Access to significant computing power could allow us to build a more advanced and faster chatbot for demonstration and initial testing.
  • Cost-Effective for Development: Using these resources for the project could be free or very low-cost, saving budget during development.
  • Good Data Control (During Project): If using the school's private systems, we maintain good control over project data.
  • Ideal for Experimentation: Allows us to experiment with more demanding AI setups without personal hardware limits during development.
  • Not a Long-Term Solution for Production: While great for the project, the school likely cannot host the live chatbot for an external client indefinitely or for free. A different hosting solution would be needed for the final product.
  • Permission & Access Dependent: We would need to confirm if we can use these resources and understand any rules or limitations.
  • Transition Required Later: We would need to plan and execute a move to a commercial provider if the chatbot goes into full production.
  • Setup & Management: May still require some technical effort to set up and manage on the school's systems, even if temporary.

3. Using a Top-Tier "AI Brain" from a Specialist Company (like ChatGPT-4o)

(We still provide our scraped event data)

This option involves "renting" time on a very advanced AI, like ChatGPT, that's run by a big company. We would feed it our collected event information to answer questions.

  • Very Smart & Capable: Access to one of the best AIs available, which understands questions very well.
  • Handles Lots of Info: Can easily work with all our event details and long conversations.
  • Always On & Ready: These services are built to be reliable and handle many users.
  • No Computer Worries for Us: The AI company handles all the powerful computers needed.
  • Quick to Get Started with AI: We can focus on our chatbot's features, not the AI's plumbing.
  • Pay-As-You-Go Costs: We'd pay based on how much we use it, which could add up with many users or questions.
  • Data Goes to a Third Party: Event details and user chats are sent to the AI company's computers.
  • Relies on Another Company: We depend on their service, pricing, and rules.
  • Less "Under the Hood" Control: We can't change the core AI itself, only how we ask it questions.
  • We Still Manage Event Data: We would still need to collect and update our event information.

4. Using an "AI Search Specialist" (like Perplexity API)

(May reduce some of our data collection work)

This is like hiring an AI that's an expert at finding information on the internet. It could answer some questions using live web data, possibly reducing our need to manually collect *everything*.

  • Gets Live Info: Can find the very latest details online, which is great for things that change often.
  • Always Up-to-Date: Can provide answers based on current web information.
  • Potentially Simpler Data Management for Us: Might mean less work for us in constantly updating some types of event details.
  • Always On & Ready: Reliable service managed by the AI company.
  • No Computer Worries for Us: They handle the technical side.
  • Pay-As-You-Go Costs: Similar to other API services, usage incurs costs.
  • Data Goes to a Third Party: User questions are sent to their servers.
  • Relies on Another Company: We depend on their service and terms.
  • Less Control Over Info Sources: While it searches online, we don't fully control where it looks or what it prioritizes compared to our own curated event list.
  • Balancing Our Data and Its Searches: We'd need to ensure it uses our specific event list (the `full_site_data.json`) effectively, and decide when it should use its web search instead. It might not automatically know everything about our specific, local events without us telling it.
